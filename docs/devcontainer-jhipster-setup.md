# Dev Container And Generation Notes

This repo uses a custom dev container instead of the devcontainer generated by JHipster. It includes Java 21, Node 24, Maven, Docker CLI, GitHub CLI, JHipster 9.1.0, and useful tools such as `rg`, `fd`, `jq`, `yq`, `fzf`, `httpie`, `shellcheck`, PostgreSQL client, and MariaDB client.

## Build The Container

1. Open this repository in VS Code.
2. Run `Dev Containers: Reopen in Container`.
3. After the rebuild, check the tools:

```bash
java -version
node --version
npm --version
jhipster --version
docker --version
gh --version
```

## Generation Command Used

JHipster 9.1.0 currently fails for this React app when translations are enabled, so this baseline was generated with translations disabled:

```bash
jhipster --defaults \
  --base-name jhipsterReactPrototype \
  --package-name com.guanhaiyu.jhipsterreactprototype \
  --client-framework react \
  --client-bundler webpack \
  --no-enable-translation \
  --skip-git
```

The same configuration is recorded in `docs/jhipster-vanilla-react-springboot.jdl`.

## First Run

Inside the dev container:

```bash
./npmw install
./npmw run backend:start
./npmw run start
```

Open the app at:

```text
http://localhost:9000
```
